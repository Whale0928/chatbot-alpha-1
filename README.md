# chatbot-alpha-1

> 회의·운영·이슈 트래킹을 한 디스코드 스레드에서 끝내는 LLM 봇.

소규모 팀의 일상 — 매주 회의 노트, 매주 이슈 정리, "그 PR 어디까지 됐어?" 같은 즉문즉답 — 을
**디스코드 채팅 한 줄 / 버튼 한 번**으로 처리하기 위해 만들었다.
미팅 메모는 자동으로 구조화된 노트가 되고, GitHub 상태는 LLM이 정리해서 던져주며,
"인프라 관련 열려있는 이슈만 보여줘" 같은 자유 지시도 그 자리에서 처리한다.

## 무엇을 제공하는가

### 1. 미팅 모드
디스코드 스레드 안에서 메모를 자유롭게 쌓고, "미팅 종료"로 마무리하면
**4가지 정리 포맷** 중 하나를 골라 LLM이 구조화된 노트를 생성한다.

- 결정+진행 / 논의 / 역할별 / 자율 (freeform)
- "**추가 요청**" 버튼으로 자연어 directive 주입 가능 — _"프론트/백엔드/기획 H3 섹션으로 묶어줘"_ 같은 스타일 지시가 같은 회의 데이터에 다른 관점으로 적용된다
- 진행 중 [중간 요약]으로 스냅샷, sticky 컨트롤 메시지로 채팅이 흘러도 종료 버튼이 항상 하단에 노출

### 2. 주간 정리
등록된 GitHub 레포의 **현재 OPEN 이슈 전체 + 지난 14일 커밋**을 한 번에 LLM에 dump해
운영 진단 마크다운을 생성한다.

- 한 줄 요약 / 주요 활동 / 닫아도 될 이슈 / 블로커 / 라벨·메타 정합성 / 다음 주 우선순위
- "**닫아도 될 이슈**" 추천은 별도 스키마 필드(`closeable[]`)로도 받아 **GitHub PATCH로 일괄 close**까지 한 번에
- follow-up 버튼: 추가 요청 / 기간 변경(14·30일) / 다시 분석 / **분석 결과를 미팅 첫 노트로 주입해 미팅 시작**

### 3. 에이전트 모드
정형화된 버튼 흐름과 별개로, **자유 자연어 지시**를 받는 모드.
> "워크스페이스에서 인프라 관련 열려있는 이슈들 가져와"
> "BE에서 지난 14일 동안 가장 큰 변경 요약해줘"

등록된 4 레포의 데이터를 병렬로 fetch해 단일 dump로 LLM에 전달, 의도 매칭은 LLM이 처리한다.

### 4. 세션이 끊기지 않는 워크플로우
미팅 finalize, 주간 분석, 에이전트 답변 — 어떤 작업이 끝나도 세션은 유지되고
[처음 메뉴]로 같은 스레드에서 다음 작업으로 이어간다. 미팅 모드 idle은 2시간, 일반은 5분.

## 기술적 포인트

- **Go + Cobra CLI**: 디스코드 봇 본체 외에 `llm-bot` / `git-bot` 검증용 서브커맨드 — 디스코드 없이 LLM 파이프라인을 stdout에서 골든 비교 가능
- **OpenAI Structured Output (`gpt-5.4-mini`, reasoning low)**: 7가지 응답 타입 모두 JSON Schema strict 강제 — 미팅 4 포맷 + interim + 주간 + 에이전트
- **환각 방지 레이어**: 응답을 원본 노트와 substring/토큰 겹침으로 1차 검증, "닫기" 같은 비가역 액션은 LLM 추천을 그대로 신뢰하지 않고 별도 schema 필드로 ground truth를 따로 받아 사용자 확인 prompt를 거친 뒤 실행
- **Discord 멘션 발송 차단**: 모든 렌더에서 `@` prefix 의도적으로 제거 — 노트는 한 명이 대신 적는 형태라 다른 사용자에게 푸시 알림이 가면 안 되는 운영 요건
- **사이클이 있는 상태 그래프**: 주간 분석 → "미팅 시작" → 미팅 모드 진입 (분석 결과가 첫 노트) → 미팅 정리 → 처음 메뉴, 같은 스레드에서 자연스럽게 흐름이 이어지도록 설계

## 스택

`Go 1.25` · `discordgo` · `openai-go v3` · `cobra` · `invopop/jsonschema` · `godotenv`

## 한계 / 의식한 트레이드오프

- 분석 단위는 **OPEN 이슈 + 14일 커밋**으로 통일했다. 처음에는 "마지막 release 이후" 같은 release 단위 분기를 BE/FE에 적용했지만, 릴리즈를 안 찍는 주가 더 많아 입력이 비어 의미가 없었다. 일관성을 우선했다.
- 에이전트 모드는 매 호출마다 4 레포 dump를 통째 LLM에 넘긴다 (~16k input tokens, ~$0.01/회). tool calling으로 효율화할 수 있지만, 단일 호출의 자유도와 단순함을 우선해 보류했다.
- 분석 대상 레포는 코드 상수로 등록(`pkg/bot/git_server.go`). Discord Guild 단위 매핑이나 첫 클릭 시 등록 같은 정교한 패턴을 검토했지만, 봇 운영 범위가 작아 코드 수정 한 번이 더 간단하다고 판단했다.
